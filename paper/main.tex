\begin{abstract}
JSON Canonicalization Scheme (RFC~8785) requires deterministic
floating-point formatting that matches ECMA-262 \texttt{Number::toString}.
We present the first empirical comparison of multiprecision
(\burgerdybvig) and fixed-width (\schubfach) digit generation inside a
specification-constrained canonicalization pipeline.  Our methodology
gates performance measurement on conformance: only implementations
passing 286{,}362 oracle vectors (generated from the V8 JavaScript
engine) and 154 structural conformance cases proceed to benchmarking.
Using permutation tests, BCa bootstrap confidence intervals, and noise-floor
documentation across 15 (CLI) and 10 (API) repeated measurements, we find that \schubfach
is 1.3--1.7$\times$ faster on number-heavy workloads in API benchmarks due to
zero heap allocation in the digit-generation path, with the advantage
attenuated to 1.1--1.4$\times$ in end-to-end CLI measurement due to
process startup overhead.  On string-dominant inputs the difference
narrows to $<$5\% where formatting cost is amortized.
Both implementations produce byte-identical output under
cross-compilation for x86-64 and arm64 (arm64 binaries verified via
QEMU user-mode emulation on x86-64 host).  We release the benchmark
infrastructure, oracle vectors, and all raw data as a reproducibility
artifact.
\end{abstract}
