\subsection{Cross-Language Validation}

All four implementations pass the full conformance matrix (308 total
cases: 77~structural tests $\times$ 4~implementations, 0~failures) and
produce byte-identical output on all oracle vectors.  2{,}000
differential fuzz cases yield zero cross-implementation divergences.

The statistical report includes four comparison classes:
\begin{itemize}
\item within-Go (\schubfach vs.\ \burgerdybvig),
\item within-Rust (\schubfach-rs vs.\ \burgerdybvig-rs),
\item same-algorithm Go-vs-Rust for \schubfach,
\item same-algorithm Go-vs-Rust for \burgerdybvig.
\end{itemize}

In CLI end-to-end measurement, Rust implementations are approximately
2--3$\times$ faster than their Go counterparts, primarily due to lower
process startup overhead (Rust native binary vs.\ Go runtime
initialization).  In API benchmarks, Rust-\schubfach is faster than
Rust-\burgerdybvig on number-dense workloads, replicating the
within-Go algorithmic ordering and confirming that the \schubfach
advantage is not language-specific.

This decomposition separates algorithmic effects from language/runtime
effects and directly addresses language-specific performance threats.
